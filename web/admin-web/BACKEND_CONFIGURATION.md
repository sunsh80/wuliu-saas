# 后端服务器配置说明

## 问题描述
前端请求被发送到 `file:///C:/api/admin/login` 而不是通过代理服务器，这表明代理配置可能没有正确生效。

## 解决方案

### 1. 更新Vite代理配置
已将代理目标从 `http://localhost:3000` 更新为 `http://192.168.2.250:3000`

### 2. 临时修复登录页面
在登录页面中直接使用完整URL以确保请求正确发送

### 3. 后端服务器配置
确保后端服务器在 `http://192.168.2.250:3000` 上运行，或配置端口转发

## 配置步骤

### 1. 后端服务器配置
确保后端服务：
- 在端口 3000 上运行
- 监听正确的网络接口（不仅是localhost）
- 配置CORS允许前端域名访问

### 2. 网络配置
- 确保 192.168.2.250 是有效的IP地址
- 检查防火墙设置
- 确保网络连通性

### 3. 前端配置
- 重启Vite开发服务器以应用新的代理配置
- 清除浏览器缓存
- 检查控制台是否有错误信息

## 测试步骤

1. 启动后端服务在 `http://192.168.2.250:3000`
2. 启动前端开发服务器: `npm run dev`
3. 访问 `http://localhost:5173/login.html`
4. 尝试登录并检查控制台日志

## 故障排除

### 如果仍然遇到问题：
1. 检查后端服务是否在指定IP上运行
2. 验证网络连通性
3. 检查CORS配置
4. 查看浏览器控制台错误信息
5. 确认Vite服务器已重启

## 注意事项
- 在生产环境中，可能需要使用不同的配置
- 确保IP地址在您的网络环境中有效
- 考虑使用环境变量管理不同环境的配置
